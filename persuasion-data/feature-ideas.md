# Argos 기능 아이디어 백로그

persuasion-review 시뮬레이션 등에서 "있었다면 이 페르소나를 설득할 수 있었을" 기능 아이디어를 기록한다. 현재 제품 기능과 구분하여 별도 관리.

발굴 프로세스: 가치제안 드래프트 과정에서 Claude가 무의식적으로 fictional 기능을 포장하는 경우가 발생 → 해당 기능을 지적·제거하고 본 파일에 맥락과 함께 저장 → 시뮬은 실제 기능만으로 재작성. 이렇게 쌓이는 백로그는 "외부자 관점에서 가장 자연스럽게 연상되는 확장"이라는 signal을 가진다.

## 2026-04-22 — ecommerce-si-ax-exec-01 (value proposition draft)

발굴 맥락: 중견 e커머스 SI(50~100명, 25+년 업력)의 AX 담당 임원 페르소나에 대한 `landing_plus_meeting` 가치제안 초안 작성 중, 미팅 섹션에서 Claude가 현재 없는 기능을 salesman 대사처럼 포함. 페르소나의 decision gate(PG·batch 실증, 개발팀장 반발, 고객사 임원 veto)에 대응하려는 압력에서 파생.

### 1. 팀·프로젝트 단위 주간 리포트 자동 생성·발송

- **현 상태**: 대시보드에 주 단위 추세 뷰는 있으나, 자동 리포트 생성·메일/슬랙 발송 기능 없음.
- **유스케이스**: "이번 주 팀별 토큰 사용량 + 호출 TOP 스킬 변동 + 반복 실패 증가 지점" 자동 집계 → PDF 또는 슬랙 다이제스트.
- **페르소나 관점 가치**: 50+명 대형 프로젝트에서 AX 임원·PM이 대시보드를 매일 들여다볼 시간 없음. 메일/슬랙에 주간 요약이 도착하면 경영 리포트 재료로 바로 활용. "먼저 측정해야 관리한다"는 카피를 **임원 레이어의 습관**으로 연결.

### 2. 코드 기능 단위 세션 분류·라벨링

- **현 상태**: 세션은 타임라인·전사로 저장되지만 기능(결제·재고·배송·마케팅) 단위 분류 없음.
- **유스케이스**: 세션에 기능 태그 부여(수동 or repo path·브랜치·파일 패턴 기반 자동 추정), 기능별 사용량·실패율·토큰 비용 집계.
- **페르소나 관점 가치**: e커머스 도메인에서 "AI가 결제 모듈에 얼마나 관여했는가" 같은 질문에 정량 답변 가능. 금융·커머스·규제 도메인에서 특히 강력. 고객사 대응과 임원 리포트 양쪽에 쓰임.

### 3. 감사 로그 전용 뷰 + 검색·필터

- **현 상태**: 세션 전사 원본은 저장되지만, 감사자에게 제시할 전용 뷰·검색·필터·readonly 공유 기능 없음.
- **유스케이스**: 시간·사용자·파일·기능 단위로 세션 검색, 감사자에게 readonly 링크 제공, 감사 기준 해시 고정으로 원본 무결성 보장.
- **페르소나 관점 가치**: 금융·공공·커머스 규제 고객사 대응에서 "AI가 뭘 건드렸는지 감사 로그 달라"는 요청에 즉답 가능. 패션 고객사 임원 veto 리스크 방어 재료.

### 4. 기능별 AI 성공/실패율 정량화

- **현 상태**: 스킬 호출 단위 반복 실패 지점은 집계되지만, "생성된 코드가 PG 연동에서 몇 번 성공·실패했는가" 같은 기능 단위 성공률은 없음.
- **유스케이스**: 세션 → 생성 코드 → 테스트/빌드/배포 결과까지 파이프라인 이벤트 수집 후, 도메인·기능 단위 성공률 계산. CI 통합 필수.
- **페르소나 관점 가치**: "AI 주도 개발이 e커머스 복잡 인터페이스(PG, batch, 외부 API)에서 실제로 작동하는가"라는 의사결정 게이트에 정량 근거 제공. **현재 이 수요에 Argos가 줄 수 있는 답이 사실상 없음** — 가장 임팩트 큰 공백.

### 5. 조직 스킬 카탈로그·라이브러리 뷰

- **현 상태**: 스킬·에이전트 호출 TOP은 있지만 "조직 내 스킬 카탈로그"(설명·담당자·품질 지표)로서의 관리 뷰 없음.
- **유스케이스**: 사내 스킬 목록, 각 스킬의 담당자·사용량·반복 실패율·개선 이력 표시. 개선 제안/리뷰 플로우.
- **페르소나 관점 가치**: "자체 솔루션 오너십 강한 개발팀장"을 감시 대상이 아닌 **"팀 지식 자산 관리자"** 로 포지셔닝할 수 있는 장치. 실무 리더 저항 감소, 오히려 도입 옹호자로 전환 가능.

### 6. 스킬·에이전트 "반복 실패 지점" 메트릭  ⚠️ 랜딩 카피 불일치

- **현 상태**: **랜딩 히어로 카피에 이미 명시**("…반복 실패 지점까지 한번에 모아보세요"). 그러나 실제 대시보드에는 구현 없음. **제품 주장과 실제 기능 간 gap**.
- **유스케이스**: 스킬/에이전트 호출 중 실패·재시도·중단 등 부정 결과 집계. 호출량 대비 실패율, 주간 추세, 구체적 실패 사례(해당 세션 링크) 드릴다운.
- **페르소나 관점 가치**: 개발팀장·시니어 관점의 "자체 솔루션 특화 스킬 중 어느 게 완성도 낮은가" 관측. 운영 품질 감지.
- **긴급도**: **높음**. 현재 랜딩 페이지를 본 방문자가 기능 존재를 전제로 기대 → 대시보드 접속 시 실망 유발. **구현 or 카피 수정 중 택일 필요**.
